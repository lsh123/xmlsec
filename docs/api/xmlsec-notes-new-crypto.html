Overview.

XML Security Library can support practicaly any cryptographic library. Currently, it has "out-of-the-box" support for OpenSSL, NSS and GnuTLS. If your favorite library is not supported yet then you can write necessary code yourself. If you and your company (university, ...) are willing to share the results of your work I would be happy to add support for new libraries to the main XML Security Library distribution.

The XML Security Library separates the cryptographic library (engine) specific code in an "xmlsec-<crypto>" library (where "<crypto>" is "openssl", "gnutls", "nss", etc.) which includes following items:

  • xmlSecCryptoApp* functions.

  • Cryptographic transforms and keys data implementation.

  • Keys store support (X509, PGP, etc.).

In this chapter, we will discuss a task of creating "xmlsec-mycrypto" library that provides support for your favorite "MyCrypto" cryptographic library.
