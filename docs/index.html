XML Security Library

XML Security Library is a C library based on LibXML2. The library supports major XML security standards:

  • XML Signature
  • XML Encryption
  • Canonical XML (part of the LibXML2)
  • Exclusive Canonical XML (part of the LibXML2)

XML Security Library is released under the MIT Licence see the Copyright file in the distribution for details.

News

  • April ?? 2023
    The XML Security Library 1.3.0 release includes the following changes:
    • Added support for MGF1 algorithms to xmlsec-openssl and xmlsec-mscng (see Interoperability report for more details).
    • Added support for RSA OAEP including MGF1 algorithms to xmlsec-nss, xmlsec-gcrypt, and xmlsec-gnutls (see Interoperability report for more details).
    • Added support for RSA PSS withtout parameters algorithms to xmlsec-openssl, xmlsec-nss, xmlsec-gcrypt, xmlsec-gnutls, and xmlsec-mscng (see Interoperability report for more details).
    • Added support for RSA PKCS 1.5 key transport to to xmlsec-gcrypt and xmlsec-gnutls (see Interoperability report for more details).
    • Added support for ECDSA signatures to to xmlsec-gcrypt (see Interoperability report for more details).
    • Switched xmlSecSize to use size_t by default (breaks ABI compatibility). Use "--enable-size-t=no" configure option ("size_t=no"" on Windows) to restore the old behaviour (note that this behaviour is deprecated and will be removed in the future.
    • Disabled by default legacy insecure algorithms (MD5 and RIPEMD160). Use "--enable-md5" and "--enable-ripemd160" configure options ("legacy-crypto" option on Windows) to re-enable these algorithms.
    • Removed support for OpenSSL 1.0.0 and LibreSSL before 2.7.0.
    • Removed deprecated functions.
    • Disabled by default support for NT4. Use "nt4=yes" configure option on Windows to re-enable it.
    • Enabled templatized output filenames in xmlsec command line utility.

  • October 25 2022
    The XML Security Library 1.2.35 release includes the following changes:
    • Migration to OpenSSL 3.0 API (based on PR by @snargit). Note that OpenSSL engines are disabled by default when XMLSec library is compiled against OpenSSL 3.0. To re-enable OpenSSL engines, use "--enable-openssl3-engines" configure flag (there will be a lot of deprecation warnings).
    • The OpenSSL before 1.1.0 and LibreSSL before 2.7.0 are now deprecated and will be removed in the future versions of XMLSec Library.
    • Refactored all the integer casts to ensure cast-safety. Fixed all warnings and enabled "-Werror" and "-pedantic" flags on CI builds.
    • Added configure flag to use size_t for xmlSecSize (currently disabled by default for backward compatibility).
    • Moved all CI builds to GitHub actions.
    • Several other small fixes (more details).
